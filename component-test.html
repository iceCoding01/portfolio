Demonstrating how the modular component system works across multiple pages

Introduction

This page demonstrates how the component system works in the portfolio. Components are reusable HTML sections that can be included in multiple pages, making it easier to maintain and update the site.

<div data-component="./components/component-name.html">
    <!-- Loading indicator (optional) -->
    <div class="loading-component">
        <div class="w-full h-40 flex items-center justify-center">
            <div class="animate-spin rounded-full h-12 w-12 border-t-2 border-b-2 border-blue-500"></div>
        </div>
    </div>
</div>

How It Works

The component system uses JavaScript to fetch and inject HTML components into the page. Here's how it works:

  1. The system looks for all elements with the data-component attribute.
  2. It fetches the HTML content from the specified path.
  3. It replaces the element's content with the fetched HTML.
  4. It dispatches a componentLoaded event when the component is loaded.

Here's the code for the component loading system:

// This script helps load HTML components across the site
document.addEventListener('DOMContentLoaded', function() {
    // Find all elements with data-component attribute
    const componentElements = document.querySelectorAll('[data-component]');
    
    // Load each component
    componentElements.forEach(element => {
        const componentPath = element.getAttribute('data-component');
        
        fetch(componentPath)
            .then(response => {
                if (!response.ok) {
                    throw new Error(`Failed to load component: ${componentPath}`);
                }
                return response.text();
            })
            .then(html => {
                element.innerHTML = html;
                
                // Dispatch event to notify that component was loaded
                const event = new CustomEvent('componentLoaded', { 
                    detail: { 
                        componentPath: componentPath,
                        element: element 
                    } 
                });
                document.dispatchEvent(event);
            })
            .catch(error => {
                console.error('Error loading component:', error);
                element.innerHTML = `
Failed to load component: ${componentPath}
`;
            });
    });
});

Benefits of Using Components

Reusability

Components can be reused across multiple pages, ensuring consistency throughout the site.

Maintainability

Make changes in one place and see them reflected across all pages that use the component.

Clean Code

Keeps your HTML files cleaner and more organized, with smaller, more focused files.

Scalability

Makes it easier to add new pages and features by reusing existing components.

Next Steps

Now that we have a working component system, here are some recommendations for further modularization:

  • Extract the header/navigation as a component
  • Extract the footer as a component
  • Create smaller, reusable UI components (buttons, cards, sections)
  • Consider using a templating system for more complex components
